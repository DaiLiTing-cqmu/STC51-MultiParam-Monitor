# STC51-MultiParam-Monitor
基于 STC51 的多参数采集与 Web 可视化系统

---

## 一、项目简介
   为解决当前临床医疗中生理信号检测设备<u>成本高、便携性不足、数据难以长期追踪</u>等局限性，本项目面向居家养老、社区医疗等轻量化、普惠型监测场景，搭建了一套从硬件采集到 Web 可视化的全链路多参数生理信号监测系统。

   系统以 **STC51** 单片机为核心，通过 **ADC** 模块实现心音、体温、环境光强、电压等多维度信号的实时采集，经自定义 FA/FB 双校验通信协议完成低延迟无线传输，将数据持久化存储至阿里云 **RDS MySQL** 云端数据库，并通过基于 **ECharts** 的 Web 前端实现动态波形可视化与历史数据查询。
   
   整套方案具备<u>低成本、易部署、可扩展的<u/>特点，能够为居家老人、慢性病患者提供便捷的日常生理监测服务，同时为社区医疗、基层健康管理提供数据支撑，助力医疗资源下沉与普惠医疗落地。

### 技术栈
#### 1.硬件层（核心使用）
- 主控芯片：STC51 单片机（8051内核）
- 数据采集：ADC 多通道采集（心音、温度、光强、电压参数）
- 通信方式：串口（UART）通信、TCP 数据传输
- 硬件设计：PCB 原理图设计（STC51 最小系统、ADC 采集电路、通信电路）
- 开发工具：Keil C51（单片机代码编译、程序烧录）

#### 2.服务端层（核心使用）
- 开发语言/框架：Node.js + Express
- 网络通信：WebSocket（实时数据推送）、Net 模块（TCP 数据流处理）
- 数据存储：阿里云 RDS MySQL（云端数据库持久化）
- 通信协议：自定义 FA/FB 双校验固定帧协议（10 字节，含帧头、数据段、校验位）

#### 3.前端层（核心使用）
- 基础技术：HTML
- 可视化：ECharts（实时波形展示、数据可视化）

#### 4.工程管理（配套使用）
- 版本控制：Git + GitHub（含.gitignore 配置）
- 部署相关：npm（Node.js 依赖管理）、云服务器部署（可选）

## 二、硬件模块说明
- `main.c`：STC51 单片机核心程序，基于 8051 内核实现 4 通道 ADC 同步采集，支持心音、体温、环境光强、生理电压等多参数实时采集，并通过 UART 串口转 TCP/IP 协议完成数据低延迟传输。
- `PCB_main.pdf`：自主设计的硬件原理图文件，包含 STC51 最小系统电路、多通道 ADC 信号调理电路、电源管理电路、串口通信接口电路等，为系统提供稳定的硬件支撑。

## 四、快速部署
### 1. 安装依赖
```bash
cd server
npm install
```
### 2. 数据库初始化
执行 sql/sensor_table.sql 创建数据表。
### 3. 修改数据库配置
修改 server/server.js 中的阿里云 RDS 配置：
```javascript

const dbConfig = {
  host: '你的阿里云RDS地址',
  port: 3306,
  user: '你的账号',
  password: '你的密码',
  database: 'sensor_data'
};
```
### 4. 启动服务
```bash
node server.js
```
### 5. WiFi链接
配置 WiFi 模块连接目标网络，建立 TCP 客户端连接服务端 IP 与端口，实现硬件采集数据向服务端的无线透传，支持断线重连机制保障传输稳定性。

### 6. 访问前端
浏览器访问：http://localhost:8081,即可查看多参数实时波形
登录阿里云账号即可检索历史数据

## 五、功能亮点
 - **多参数 ADC 实时采集**：基于 STC51 单片机实现 4 通道同步采集，支持心音、体温、环境光强、电压等多维度生理/环境信号，采样稳定、时序可控。
 - **高可靠自定义通信协议**：设计 FA/FB 双校验固定帧协议（10 字节），包含帧头标识、数据段与校验位，数据传输还原率达 **99%+**，有效抵御传输干扰。
 - **低延迟 Web 可视化**：基于 WebSocket 实现服务端-前端实时数据推送，结合 ECharts 绘制动态波形，支持多参数切换、实时告警与毫秒级响应。
 - **云端数据持久化与查询**：依托阿里云 RDS MySQL 实现数据云端存储，支持历史数据分页查询、时间范围筛选，为长期健康监测提供数据支撑。
